配置用户访问权限

Grid Engine 系统有以下四种用户：
■ 管理人员。 管理人员可以完全操作 Grid Engine 系统。默认情况下，主控主机和具有
队列实例的主机的超级用户具有管理人员权限。

■ 操作员。 操作员除了不能添加、删除或修改队列以外，可以与管理人员一样执行许多
相同的命令。

■ 拥有者。 队列拥有者被限定为只能对其所拥有的队列进行暂停和恢复，或者禁用和启
用。 这些权限对于成功使用 qidle 是有必要的。用户通常声明为其台式工作站上的
队列实例的拥有者。

■ 用户。 用户具有某些访问权限，但用户不具
有管理群集或队列的权限。
用 QMON 配置管理人员帐户
在 “QMON Main Control” 窗口中，单击 “User Configuration” 按钮

从命令行配置管理人员帐户
要从命令行配置管理人员帐户，请键入以下命令并带上适当的选项：
# qconf options
以下为可用的选项：
■ qconf -am user-name [,...]
-am 选项（添加管理人员）向 Grid Engine 系统管理人员列表中添加一个或多个用
户。默认情况下，所有受信主机的 root 帐户都是 Grid Engine 系统管理人员。有关
详细信息，请参阅第 20 页中的 “关于主机和守护进程” 。
■ qconf -dm user-name [,...]
-dm 选项（删除管理人员）删除 Grid Engine 系统管理人员列表中的指定用户。
■ qconf -sm
-sm 选项（显示管理人员）显示所有 Grid Engine 系统管理人员的列表。



从命令行配置操作员帐户
要从命令行配置操作员帐户，请键入以下命令并带上适当的选项：
# qconf options
以下为可用的选项：
■ qconf -ao user-name[,...]
-ao 选项（添加操作员）向 Grid Engine 系统操作员列表中添加一个或多个用户。
■ qconf -do user-name[,...]
-do 选项（删除操作员）从 Grid Engine 系统操作员列表中删除指定的用户。
■ qconf -so
-so 选项（显示操作员）显示了所有 Grid Engine 系统操作员的列表。
